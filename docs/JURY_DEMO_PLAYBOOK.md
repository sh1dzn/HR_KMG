# JURY Demo Playbook

## Цель демо (6-8 минут)
Показать, что продукт не просто “генерит цели”, а закрывает полный цикл:
постановка -> согласование -> исполнение -> контроль рисков -> аналитика.

## Сценарий показа
1. **Сотрудник: генерация и доработка**
- Открыть `Генерация целей`.
- Сгенерировать 3-5 целей с каскадированием от руководителя.
- Показать SMART-оценку и источники (RAG по ВНД).

2. **Сотрудник: исполнение через Kanban**
- Открыть `Мои цели` и переключить режим `Канбан`.
- Перетащить цель между колонками (например `draft -> in_progress`).
- Нажать `Выполнить` и показать переход в `done`.

3. **Руководитель: контроль качества и согласование**
- В `Сотрудники` открыть workflow конкретной цели.
- Выполнить действие `Утвердить` или `На доработку`.
- Показать историю событий и комментарии.

4. **Администратор: аналитика и риски**
- Открыть `Дашборд`.
- Показать зрелость подразделений, тренды по кварталам и блок рисков.

5. **AI-ассистент + MCP**
- В чат задать вопрос по проблемным подразделениям/топ-сотрудникам.
- Показать, что ответ в Markdown и с токен-метриками.
- Дополнительно открыть `GET /api/mcp/public/info` и показать, что MCP доступен как remote endpoint для внешних AI-клиентов.

## Что подчеркивать для жюри
- **End-to-end продукт**: есть полный процесс, а не отдельный AI-виджет.
- **Role-based governance**: сотрудник/руководитель/админ видят разное и управляют в рамках прав.
- **Инженерная зрелость**: автотесты, CI, API-контракты, воспроизводимый деплой.
- **Практическая польза**: снижение слабых целей, прозрачность исполнения, раннее обнаружение рисков.

## Быстрый fallback, если API OpenAI недоступен
- Показать эвристическую SMART-оценку и дашборд.
- Показать workflow и kanban-движение цели.
- Показать данные и фильтры по ролям.
